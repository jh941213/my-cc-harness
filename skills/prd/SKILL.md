사용자의 아이디어를 인사이트 중심으로 분석하여 PRD를 생성합니다.
기술 상세는 후속 `/spec`에서 다룹니다.

입력: $ARGUMENTS

## 실행 프로세스

### Step 1: 시장 리서치 (서브에이전트 3개 병렬)

다음 3개 리서치를 **반드시 병렬 서브에이전트**로 동시에 실행:

**서브에이전트 A: 경쟁사 & 유사 제품** (subagent_type: general-purpose)
- Tavily로 유사 제품/서비스 5-10개 검색
- 각 제품의 핵심 강점, 약점, 가격, 타겟 정리
- 핵심: **모든 경쟁사가 공통으로 놓치는 것** 식별

**서브에이전트 B: 오픈소스 & 기술 생태계** (subagent_type: general-purpose)
- Exa로 GitHub 유사 프로젝트 검색
- 스타 수, 기술 스택, 미해결 Issue(사용자 니즈) 분석
- 핵심: **오픈소스의 한계점 = 우리의 기회** 도출

**서브에이전트 C: 사용자 페인포인트 & 시장 동향** (subagent_type: general-purpose)
- Tavily로 Reddit, HN, 개발자 커뮤니티 불만/니즈 검색
- "사용자가 말하는 것 vs 실제로 원하는 것" 구분
- 핵심: **반복되는 불만 패턴** 및 **아직 해결 안 된 니즈** 추출

### Step 2: 리서치 브리핑 + 1차 질문

서브에이전트 결과를 핵심만 요약하여 사용자에게 보여주고,
AskUserQuestion으로 **라운드 1: 진짜 문제 파기** 시작.

### Step 3: 인사이트 인터뷰 (5라운드)

AskUserQuestion으로 반복 질문:

**라운드 1**: 진짜 문제 파기 — Why 5번, 직접 계기, 현재 대안의 한계
**라운드 2**: 숨겨진 가정 뒤집기 — 의외의 사용자, 반대 접근, "딱 하나만 만들면?"
**라운드 3**: 사용자 행동의 진실 — 30초 가치 느끼기, 이탈 이유, 말 vs 행동
**라운드 4**: 기회와 경계 — 경쟁사 공백, 안티-기능, 실패 시나리오
**라운드 5**: 우선순위의 진실 — 모순 지적, 트레이드오프 강제, 6개월 후 회고

### Step 4: 인사이트 합성

- 킬러 인사이트 3-5개 정리
- 가정 위험도 매트릭스 작성
- 차별화 포지셔닝 한 문장 정의
- 경쟁사 Feature Matrix + 공백 분석

### Step 5: PRD 생성

PRD.md를 생성합니다. 저장 위치는 프로젝트 디렉토리가 있으면 `./PRD.md`.

### Step 6: 자가 검증 + /spec 연결

완료 후 안내: "PRD.md가 생성되었습니다. 기술 상세는 `/spec PRD.md 기반으로 작성해줘`로 이어가세요."

## 핵심 인터뷰 기법

- **반론 제시**: "근데 경쟁사 X는 이렇게 해서 성공했는데요?"
- **모순 탐지**: 라운드 간 답변이 다르면 즉시 지적
- **구체화 강제**: "많은 사용자" → "몇 명?" / "빠르게" → "몇 초?"
- **침묵 활용**: "잘 모르겠다" = 검증 필요 항목으로 태깅
- **리서치 무기화**: 모든 질문에 데이터/사례 첨부

## 중요 규칙

- 리서치는 반드시 서브에이전트 병렬 실행
- 질문은 AskUserQuestion 도구만 사용
- 기술 상세는 PRD에 넣지 않음 → /spec의 영역
- 검색: 웹 → Tavily MCP, 코드 → Exa MCP, 문서 → Context7 MCP
