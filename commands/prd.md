사용자의 아이디어를 인사이트 중심으로 분석하여 PRD를 생성합니다.
기술 상세는 후속 `/spec`에서 다룹니다.

입력: $ARGUMENTS

## 실행 프로세스

### Phase 0: 입력 수신 + 복잡도 게이트

**먼저 확인**: `PRD.partial.md`가 존재하면 → "이전 세션을 이어서 진행할까요?" 질문.
존재하지 않으면 새로 시작.

사용자 입력을 받으면 **복잡도를 판정**하고 수렴 보드를 초기화:

```
Low  (단순 기능/도구/CLI): Phase 1 스킵, Phase 2를 2라운드로 축소
Mid  (새 모듈/서비스/라이브러리): Phase 1 축소(서브에이전트 1개), Phase 2를 3라운드
High (새 제품/SaaS/플랫폼): 전체 플로우 실행
```

복잡도를 판정한 뒤 사용자에게 고지. 조정 요청 시 수용.

**수렴 보드 초기화**:
```
| 차원 | 상태 | 잔여 |
|------|------|------|
| 용어 정렬 | 🔴 | - |
| 구조 (W6H) | 🔴 | 7개 미확인 |
| 깊이 (가정 검증) | 🔴 | - |
| 일관성 (모순) | 🔴 | - |
| 견고성 (반론) | 🔴 | - |
| 시장 컨텍스트 | 🔴 | - |
```

---

### Phase 1 + Phase 2 라운드 1: 병렬 시작

**동시에 실행** (라운드 1은 리서치 결과 불필요):

#### [병렬 A] 시장 리서치 서브에이전트 (background: true)

복잡도별 적응:
- **High**: 서브에이전트 3개 병렬
- **Mid**: 서브에이전트 1개
- **Low**: 스킵

**서브에이전트 A: 경쟁사 & 유사 제품** (subagent_type: general-purpose)
- Tavily + Gemini CLI로 유사 제품/서비스 5-10개 검색 (교차 검증)
- 핵심: **모든 경쟁사가 공통으로 놓치는 것** 식별

**서브에이전트 B: 오픈소스 & 기술 생태계** (subagent_type: general-purpose)
- Exa로 GitHub 유사 프로젝트 + Gemini CLI로 기술 트렌드 보완
- 핵심: **오픈소스의 한계점 = 우리의 기회** 도출

**서브에이전트 C: 사용자 페인포인트 & 시장 동향** (subagent_type: general-purpose)
- Tavily + Gemini CLI로 Reddit, HN, 커뮤니티 불만/니즈 검색
- 핵심: **반복되는 불만 패턴** 및 **미해결 니즈** 추출

#### [병렬 B] 라운드 1: 용어 정렬 + 구조 스캔 (메인 세션)

리서치 결과 없이 즉시 시작:
- **[용어]** 모호한 용어 탐지 → 정의 강제
- **[구조:W6H]** 7개 차원 중 비어 있는 것을 AskUserQuestion으로 질문
- **수렴 조건**: 핵심 용어 lock + W6H 7/7

### Phase 2: 적응적 인터뷰 (라운드 2부터, 메인 세션)

리서치 결과 합류 후 계속. 하드캡: Low 2라운드, Mid 3라운드, High 5라운드.

| 라운드 | 프레임워크 | 목적 | 수렴 조건 |
|--------|-----------|------|-----------|
| 2 | 데카르트 + White/Red Hat | 방법적 회의 + 깊이 | 핵심 차원 🟢, 🟡≤2 |
| 3 | 에렌쿠스 + 조하리 + Black/Green Hat | 모순 탐지 + 맹점 | 치명적 모순 0개 |
| 4 | 변증법 + 해석학적 순환 + Yellow/Blue Hat | 반론 단련 + 정합성 | 반론-대응 쌍 기록 |
| 5 | Blue Hat (High만) | 우선순위의 진실 | 수렴 보드 전체 🟢 |

Low → 라운드 2 후 종료, Mid → 라운드 3 후 종료.

#### 인터뷰 운영 규칙

- **질문 태그**: `[구조:Who]`, `[회의:가정]`, `[모순]`, `[반론]`, `[정합성]`, `[맹점]`
- **레이어 병합**: 한 질문 세트에서 여러 레이어 동시 처리
- **Graceful Exit**: 매 라운드 후 "계속/정리" 옵션
  - 중간 이탈 → `PRD.partial.md` 저장 (수렴 보드 + 완료 라운드 포함)
- **"모르겠다"**: 1회→선택지 제시, 3회→리서치/프로토타입 제안, "나중에"→TODO
- **회귀 상한**: 동일 차원 2회, 전체 3회 → 초과 시 TODO 태깅
- 수렴 보드: 매 라운드 시작 시 변경 차원만 업데이트

---

### Phase 3-5: prd-planner 서브에이전트에 위임

인터뷰 완료 후, **prd-planner 서브에이전트를 spawn**하여 합성 + 작성을 위임:

```
Agent 호출:
  subagent_type: prd-planner
  prompt: |
    아래 데이터를 기반으로 PRD를 생성해주세요.

    ## 복잡도: [Low/Mid/High]

    ## 리서치 결과
    [서브에이전트 A/B/C 결과 요약]

    ## 인터뷰 결과
    [라운드별 질문-답변 요약]

    ## 수렴 보드 현재 상태
    [수렴 보드 테이블]

    PRD를 ./PRD.md에 저장해주세요.
```

prd-planner가 수행하는 작업:
- Phase 3: Six Hats 교차 분석, 경쟁 지형, 가정 위험도, 차별화, 킬러 인사이트
- Phase 4: MVP 경계선, 위험 분석, 성공 지표, /spec 연결
- Phase 5: PRD.md 작성 (수렴 보드 부록 포함)

---

### Phase 6: 자가 검증 (메인 세션)

prd-planner가 생성한 PRD.md를 읽고 최종 검증:
- 모든 기능에 뒷받침 인사이트가 있는가?
- 위험한 가정 Top 3 명시?
- 안티-기능 정의?
- 실질 지표 (허영 지표 아닌)?
- /spec 넘길 항목 목록?
- 수렴 보드 전체 🟢 또는 TODO?

검증 실패 시 → prd-planner에 수정 요청 (resume).
완료 후: "PRD.md가 생성되었습니다. `/spec PRD.md 기반으로 작성해줘`로 이어가세요."

## 중요 규칙

- Phase 1 리서치 + Phase 2 라운드 1은 **반드시 병렬 시작**
- Phase 3-5는 **prd-planner 서브에이전트에 위임** (메인 컨텍스트 보호)
- 질문은 AskUserQuestion만 사용
- **절대 가정하지 않음** — 모호하면 질문
- 리서치 결과를 질문의 무기로 활용
- 모순 발견 시 정면 지적
- 기술 상세(스택, 아키텍처, API)는 /spec 영역
- 검색: 웹 → Tavily MCP / Gemini CLI, 코드 → Exa MCP, 문서 → Context7 MCP
- Graceful Exit: 언제든 중간 이탈, PARTIAL PRD 저장
